## Review

- Correct: Protected routes wait for restoration and redirect unauthenticated users (`frontend/src/features/auth/components/RequireAuth.tsx:6-16`); refresh requests use a non-intercepted client and per-request retry marker (`frontend/src/lib/api-client.ts:16-50`).

- Blocker — High: A clean checkout cannot make credentialed SPA requests. The committed CORS configuration has no allowed origin (`backend/src/InventoryFlow.Api/appsettings.json:3-5`), while the local development configuration containing `http://localhost:5173` is gitignored and untracked. `Program.cs` applies this empty list (`backend/src/InventoryFlow.Api/Program.cs:34-41`). Smallest remediation: commit a non-secret development CORS configuration (and unignore it) with `http://localhost:5173`, retaining explicit production origins via deployment config.

- Blocker — High: Required auth/replay integration coverage is absent. There is no `AuthenticationEndpointsTests.cs`; the integration project has no Testcontainers dependency (`backend/tests/InventoryFlow.IntegrationTests/InventoryFlow.IntegrationTests.csproj:8-15`), and the factory only points to a nonexistent SQL Server host without migration/setup (`backend/tests/InventoryFlow.IntegrationTests/Api/InventoryFlowApiFactory.cs:25-33`). The passing test suite therefore does not attest registration, cookie flags, `/me`, rotation, replay, concurrency, or logout. Smallest remediation: add the specified SQL Server Testcontainers fixture and focused endpoint tests.

- Note — Medium: `getCurrentUser` calls `authClient`, which never attaches the in-memory bearer token (`frontend/src/features/auth/auth-api.ts:24-27`), but `/api/auth/me` requires authorization (`backend/src/InventoryFlow.Api/Controllers/AuthController.cs:63-72`). Any caller of this helper always receives 401. Smallest remediation: send this request through `apiClient` (or explicitly attach `Authorization`).

- Note — Medium: An in-flight refresh can restore local authentication after logout. Refresh success unconditionally invokes `setSession` (`frontend/src/lib/api-client.ts:19-24`), whereas sign-out clears state without invalidating or aborting that promise (`frontend/src/components/layout/Topbar.tsx:41-47`). Smallest remediation: use a session-generation/abort guard so refresh responses started before logout cannot commit state afterward.

- Note — Low: Requested-route restoration drops query and fragment because the guard saves only `location.pathname` (`frontend/src/features/auth/components/RequireAuth.tsx:15`). Smallest remediation: retain `${pathname}${search}${hash}` and continue applying the existing in-app-path validation.

- Note — Low: Dynamic form errors are neither announced nor associated with the affected control (`frontend/src/features/auth/pages/LoginPage.tsx:62`, `frontend/src/features/auth/pages/RegisterPage.tsx:71`). Smallest remediation: add an `aria-live`/`role="alert"` error region and `aria-describedby`/`aria-invalid` on invalid fields.

Blocking issues exist.
