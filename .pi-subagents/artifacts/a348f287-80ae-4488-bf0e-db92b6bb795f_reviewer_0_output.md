- **High:** Logout is not atomic with refresh rotation. `LogoutAsync` reads a token, separately queries active family tokens, then saves without a transaction (`backend/src/InventoryFlow.Infrastructure/Authentication/IdentityAuthenticationService.cs:90-96`). A concurrent refresh can rotate after logout has read the active family set but before logout saves, leaving the newly issued replacement active; a holder of that replacement can continue refreshing after logout. Smallest fix: perform lookup, family revocation, save, and commit in a serializable transaction wrapped by `CreateExecutionStrategy().ExecuteAsync`, mirroring `RefreshAsync`; add a coordinated concurrent logout-vs-refresh integration test. The current logout test is sequential (`AuthenticationEndpointsTests.cs:97-109`) and cannot detect this race.
