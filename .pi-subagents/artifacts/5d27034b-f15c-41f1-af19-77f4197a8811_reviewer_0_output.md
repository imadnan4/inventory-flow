A replayed old token therefore returns 401 while its active replacement remains usable. **Smallest safe remediation:** persist the family revocations before committing; add a replay/concurrent-refresh test proving no active family token remains.

- **Blocker — Critical:** Refresh rotation combines a user transaction with SQL Server retry-on-failure. `UseSqlServer(...EnableRetryOnFailure())` is configured (`backend/src/InventoryFlow.Infrastructure/DependencyInjection/InfrastructureServiceCollectionExtensions.cs:69-71`), while refresh starts its own serializable transaction (`.../IdentityAuthenticationService.cs:47`). EF Core’s configured retrying execution strategy rejects user-initiated transactions unless the entire unit is run through `CreateExecutionStrategy().ExecuteAsync(...)`; active refreshes can consequently fail at runtime rather than rotate. **Smallest safe remediation:** wrap transaction creation, lookup, revocation/replacement, save, and commit in the SQL Server execution strategy delegate, retaining serializable isolation.
